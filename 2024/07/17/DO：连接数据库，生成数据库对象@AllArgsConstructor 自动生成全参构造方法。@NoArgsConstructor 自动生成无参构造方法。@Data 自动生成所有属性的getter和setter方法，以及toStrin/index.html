学习笔记

  1. DO：连接数据库，生成数据库对象

    • @AllArgsConstructor: 自动生成全参构造方法。

    • @NoArgsConstructor: 自动生成无参构造方法。

    • @Data: 自动生成所有属性的getter和setter方法，以及toString、equals和hashCode方法。

    • @TableName("device_area"): 这是MyBatis-Plus提供的注解，用于指定数据库表名为 device_area，使得该类可以映射到名为 device_area 的数据库表。

    • @TableField("parent_id"): 指定该字段对应数据库表中的 parent_id 列。

    • @ApiModelProperty(value = "父id"): 用于生成API文档，指定该属性在API文档中的描述。

  2. VO：用于前端和后端的数据传递。

    • @TableId: 这是MyBatis-Plus提供的注解，用于指定该字段是数据库表的主键，表明 id 字段对应数据库表中的主键列。
    • @Accessors(chain = true): 这也是Lombok注解，用于生成链式setter方法，可以实现 setXxx().setYyy().setZzz() 这样的链式调用。

  3. Service

    • @Override: 表示该方法是重写自父类或接口的方法。

    • @Transactional(rollbackFor = Exception.class): 表示该方法在执行时开启事务，当发生任何异常时都会回滚事务，即保证了操作的原子性和一致性。

      @Transactional(rollbackFor = Exception.class)

      意味着如果方法中抛出了任何 Exception 及其子类的异常，事务都应该回滚。这样可以确保在遇到意外情况时，数据库不会处于一个不一致的状态。

      BeanUtils.copyProperties(e, deviceArea); deviceAreaDOS.add(deviceArea);

- `vos.stream()`: 将 `vos` 列表转换为一个流，使得可以进行流式操作。
- `.map(DeviceAreaRequestVO::getName)`: 使用 `map` 操作将每个 `DeviceAreaRequestVO` 对象映射为其 `name` 属性。这里使用了方法引用的方式，`DeviceAreaRequestVO::getName` 等效于 `vo -> vo.getName()`。
- `.collect(Collectors.toSet())`: 使用 `collect` 操作将流中的元素收集到一个 `Set` 集合中。由于 `Set` 不允许重复元素，所以这样可以确保 `names` 集合中的元素是唯一的。

    • public static <K, V> List<V> copyList(List<K> source, Class<V> zclass)


  方法名为 copyList，接受两个参数：

  `source`: 源列表，类型为 `List<K>`，其中 `K` 是源列表中元素的类型。

  `zclass`: 目标类的 `Class` 对象，表示目标列表中元素的类型。

    • CollUtil.isEmpty() 的作用类似于 CollectionUtils.isEmpty()（Apache Commons Collections 中的方法）或者 Collections.isNullOrEmpty()（Guava 中的方法），用来判断集合是否为空，即集合为 null 或者没有元素。

private List<DeviceAreaRequestVO> convertToRequestVO(List<DeviceAreaExcelVO> excelVOs) {
    return excelVOs.stream()
            .map(excelVO -> {
                DeviceAreaRequestVO requestVO = new DeviceAreaRequestVO();
                requestVO.setName(excelVO.getName());
                requestVO.setCode(excelVO.getCode());
                requestVO.setParentName(excelVO.getParentName());
                requestVO.setLevel(excelVO.getLevel());
                return requestVO;
            })
            .collect(Collectors.toList());
}
//注意括号的作用域：首先把excelVOs转换为流，进行操作，对其中的每一个excelVO对象进行大括号里面的操作，将excelVO的属性复制给requestVO，然后返回requestVO，这个返回的requestVO会进行.collect操作，然后最后返回的是requestVO的集合。
